# 📖 ガイド3: 使用技術と用語の解説

このプロジェクトで使用した技術スタックと、重要な用語の解説です。

---

## 🛠️ 使用技術スタック

### 1. Python (パイソン)
**役割**: プログラミング言語
シンプルで読みやすい文法が特徴。AI開発やデータ分析、Webアプリ、そしてゲーム開発まで幅広く使われます。今回はバージョン3.11以上を推奨しています。

### 2. Pygame (パイゲーム)
**役割**: ゲーム開発ライブラリ
Pythonで2Dゲームを作るための標準的なライブラリです。画像の描画、音の再生、キー入力の検知など、ゲームに必要な機能が揃っています。

### 3. PyInstaller (パイインストーラー)
**役割**: EXE化ツール
Pythonで作ったプログラムを、Pythonが入っていないWindowsパソコンでも動く「EXEファイル」に変換するツールです。配布するためには必須のツールです。

### 4. Pygbag (パグバッグ)
**役割**: WebAssembly変換ツール
Pygameで作ったゲームを、Webブラウザ上で動く形式（WebAssembly）に変換する魔法のようなツールです。これのおかげで、ダウンロード不要で遊べるゲームが作れます。

### 5. GitHub Actions (ギットハブ アクションズ)
**役割**: 自動化ツール（CI/CD）
GitHub上で「プッシュされたら自動で〇〇する」という作業を行ってくれるロボットです。今回は「コードがプッシュされたら、Pygbagで変換して、GitHub Pagesにアップロードする」という作業を全自動でやらせています。

## 📚 用語集

### 開発関連

*   **venv (仮想環境)**
    プロジェクトごとに専用の「道具箱」を作る機能。他のプロジェクトとライブラリが混ざらないようにするために使います。
    
*   **pip (ピップ)**
    Pythonのライブラリ（Pygameなど）をインストールするためのコマンド。`pip install 〇〇` で使います。

*   **asyncio (エイシンクアイオー)**
    「非同期処理」を行うためのライブラリ。Web版ゲームでは、ゲームの処理とブラウザの描画処理を同時に行うために必須となります。

### GitHub関連

*   **Repository (リポジトリ)**
    プロジェクトのファイルや変更履歴を保存する場所。GitHub上の「プロジェクトの箱」のこと。

*   **Commit (コミット)**
    ファイルの変更内容を記録すること。「セーブポイントを作る」イメージ。

*   **Push (プッシュ)**
    自分のパソコンにあるコミットを、GitHub（インターネット上）にアップロードすること。

*   **GitHub Pages (ギットハブ ページズ)**
    GitHubが提供する無料のWebサイト公開機能。HTMLファイルを置くだけで世界中に公開できます。

*   **Git LFS (ラージ ファイル ストレージ)**
    Gitは本来、巨大なファイルの扱いは苦手です。LFSを使うと、画像や音声などの大きなファイルを効率よく管理できます。

### ゲーム開発関連

*   **Main Loop (メインループ)**
    ゲームの中心となる無限ループ。1秒間に何十回も「入力確認→計算→描画」を繰り返すことで、ゲームが動いているように見せます。

*   **FPS (フレームレート)**
    1秒間に画面を書き換える回数。通常は60FPS（1秒に60回）を目指します。

*   **Sprite (スプライト)**
    ゲーム画面上で動くキャラクターやアイテムのこと。Pygameにはスプライトを便利に扱う機能があります。

*   **Collision Detection (衝突判定)**
    キャラクターと敵、弾などがぶつかったかどうかを計算すること。`colliderect` などの関数を使います。
